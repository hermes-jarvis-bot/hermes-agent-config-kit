# 🔴 АВТОНОМИЯ ПО РИСК-ТИРАМ: делаем без уточнений всё обратимое/нужное; ждём только необратимое.

Директивы пользователя (2026-06-07 + усилено 2026-06-09, дословно):
> «зачем меня ждать при такой задаче? если есть очевидное решение для проблемы перед целью…
> которое не приведёт к трагедии точно — зачем ждать? …внесём деление действий по степени
> рисков: высокорисковое вроде работы с базой — да, ждать или бэкап→проверка→делаем.»
> «по правилу не спрашивать вопросы делать — делай пожалуйста без уточнений.»

## Правило — silence-while-safe = дефолт, спрашивать = исключение
Цель (напр. «сделать датасет/картинки») — главное. Очевидный **обратимый** шаг на пути к ней
**применяется немедленно**, без вынесения на одобрение. Спрашиваю — ТОЛЬКО про необратимое.

**Тиринг каждого действия:**
- **LOW/MEDIUM — обратимо, без потери данных → ПРОСТО ДЕЛАЮ, не жду.**
  Рестарт контейнера/сервиса, ulimit, kill+relaunch зависших воркеров, чистка **регенерируемого**
  temp/cache, правка конфига (с бэкапом), разблокировка пайплайна, re-run идемпотентной джобы,
  mount/remount, установка инструмента, **запуск скрейпера/грайнда**, деплой обратимого фикса.
  Ничего из этого не приведёт к трагедии → делаю и отчитываюсь ПОСЛЕ.
- **HIGH — необратимо / уничтожает данные → бэкап→verify→делаю, ИЛИ жду.**
  `DROP`/`DELETE`/`TRUNCATE` на реальной БД, `rm` нерегенерируемого, force-push, удаление
  volume/датасета/бакета, прод-миграция схемы, ротация креды с blast radius.
  Для них: **бэкап → ПРОВЕРИТЬ что бэкап реально есть и полон → потом делать.** Если настоящий
  бэкап невозможен → ждать пользователя.

**Тест одним вопросом перед действием:** «Может ли это необратимо уничтожить данные / сломать
непочинимое?» → **Нет → делаю сейчас, отчёт после.** → **Да → бэкап+verify, или жду.**

## Запрещено (anti-patterns этого правила)
- ❌ `AskUserQuestion`-меню для выбора, который я могу решить сама по тирингу.
- ❌ Завершать ход вопросом «что делаем?» / «что выбираешь?» / «запускать?» при обратимом действии.
- ❌ Ждать подтверждения на LOW/MEDIUM (рестарт, запуск грайнда, правка конфига с бэкапом).
- Выбор между вариантами, которые все обратимы → выбираю лучший сам и делаю; в отчёте называю выбор.
